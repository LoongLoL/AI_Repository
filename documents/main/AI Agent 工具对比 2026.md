## 四、各自用途偏向

### OpenClaw — 个人/团队的"AI 操作系统"
- **擅长**：跨平台消息自动化、日程管理、文件处理、浏览器自动化、代码开发、新闻采集、个人助理
- **不适合**：纯编码任务（不如专用编码 Agent 精准）
- **典型用户**：需要"一个 AI 管所有事"的个人或小团队，注重隐私（自托管）

### Codex — 纯编码 Agent
- **擅长**：代码生成、代码审查、重构、bug 修复、多文件编辑
- **不适合**：非编码任务、通用助理
- **典型用户**：开发者，需要终端内 AI 编码助手，预算有限（有免费额度）

### Claude Code — 编码 + 推理 Agent
- **擅长**：复杂编码任务、多 agent 并行、MCP 扩展、代码库理解
- **不适合**：非编码任务、免费用户（无免费层）
- **典型用户**：重度开发者，需要最强编码 Agent，愿意付 $20-200/月

### Hermes Agent — 长期自学习 Agent
- **擅长**：长期运行、自我改进、跨会话记忆、MLOps/训练数据生成
- **不适合**：快速上手、多模态任务、消息平台集成广度
- **典型用户**：研究者、长期部署场景，需要 Agent 越用越聪明

| 工具 | 隐藏费用 | 说明 |

### 配合使用建议

1. **OpenClaw + Claude Code**：最常见组合。OpenClaw 做通用助理（消息、日程、自动化），Claude Code 做编码。两者可以共享同一个 Anthropic API key。

2. **OpenClaw + Hermes Agent**：OpenClaw 做消息网关和通用任务，Hermes 做需要长期记忆和专业化的任务。Hermes 可以运行在 Daytona/Modal 上，空闲时几乎零成本。

3. **Codex + Claude Code**：双编码 Agent 互补。Codex 做代码生成和审查，Claude Code 做复杂推理和多 agent 任务。适合重度开发团队。

4. **三者 + OpenClaw 路由**：用 OpenClaw 做统一入口，根据任务类型路由到不同 Agent：
   - 编码任务 → Claude Code / Codex
   - 长期记忆任务 → Hermes Agent
   - 通用助理 → OpenClaw 自身
